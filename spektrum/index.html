spektrum

Live-coding instruments live on a spectrum. At one end: tiny languages built from scratch on strong functional primitives — you play the abstraction. At the other: a coding agent that rewrites the instrument while it plays — you play the conversation. This lab builds one working environment that spans the whole spectrum, and an ICLC talk about what we found along the way.

Sound on. Everything runs in the browser — Web Audio synthesis, drums generated from first principles (no sample files), GLSL fragment shaders for visuals. No build step, no framework.

The spectrum

Pick a point. Each stop below is a fully playable station in the lab — and crucially, they are three views of the same running system, not three separate toys.

One substrate, three skins

Underneath all three stations sits one engine and one project: 

scene/pattern.js    the text program        (station I edits this)
scene/graph.json    the node patch          (station II serializes here)
scene/visual.glsl   the fragment shader     (any station)
docs/REFERENCE.md   the manual — read by humans and by the agent

A shared clock measures time in cycles (bars). A pattern is a pure function from a time span to events; the scheduler queries it just ahead of the playhead. The node patcher compiles onto the same Web Audio graph and the same clock. The agent gets Claude-Code-style tools — list_files, read_file, write_file, run_file — over those same files. Switch stations mid-performance; the music never stops.

Station I — primitives

The minimal end: a pattern language rebuilt from scratch in a few hundred readable lines, in the Tidal/Strudel tradition. The core idea is one type — Pattern = (begin, end) → events — and everything else is function composition: fast, rev, every, euclid, degrade. Even randomness is a pure function of time, so a "random" pattern is identical on every loop and every machine.

Try the mini-notation. This widget queries the actual pattern engine — the same code the lab plays:

try: bd*4, [~ hh]*4 · bd(3,8) sn · bd!3 <sn cp> · [bd sn]*2 hh?

Two cycles are shown; layers stack vertically. Note how <a b> alternates per cycle and (3,8) distributes hits euclideanly — rhythm as arithmetic.

Station II — nodes

The middle of the spectrum, where my own history lives (NodeBox, visual dataflow). Sequencers, samplers, synths, filters, LFOs and a macro value knob, patched with typed cables: solid for audio, dashed for events, dotted for modulation. Parameter drags apply live without recompiling; only re-wiring rebuilds the graph, and the transport keeps running through it. The patch is data — it round-trips through scene/graph.json, which means the agent can read your patch, reason about it, and rewrite it as a file.

Station III — the agent

The far end: live coding in the age of agentic AI. Station III is a miniature Claude Code whose repository is the scene and whose deploy target is right now: it lists files, reads the same manual you do, writes a change, runs it, hears the error, fixes it — while the previous version keeps playing underneath. Laying the tracks while the train is running.

The model runs behind a ~60-line Cloudflare Worker (worker/) that holds the API key; the browser never sees it. No key at hand? The lab ships a recorded session whose tool calls execute for real — only the model's words are canned, and the UI says so.

The interesting questions are not technical. When the agent types, what is the performance? Our answer, defended in the talk: the performer's attention moves up one level — from editing text to curating intentions, the same move that took us from soldering to patching to typing. It is still liveness; the risk surface just changes shape (the model can be wrong in more interesting ways than a typo).

Ergonomics: what survived across the whole spectrum

Building the same instrument three times is an ergonomics experiment with built-in controls. The principles that held at every point of the spectrum:

  • Never go dark. Errors replace nothing: a bad eval, a broken shader, a malformed patch — the previous state keeps playing while you fix it. The agent gets the same guarantee, and reads the error back through its tools.
  • One gesture to sound. Play runs the current scene; every demo scene is audible in one click. Cold-start silence kills momentum on stage.
  • The panic key is sacred. esc hushes everything, everywhere, always — even mid-agent-turn.
  • Show what just ran. The evaluated block flashes; tool calls appear as chips; the beat is visible in the header. Feedback is the instrument's soul.
  • Deterministic randomness. Chance operators hash musical time, so takes are reproducible and loops don't drift between machines.
  • One manual. Human and agent read the same docs/REFERENCE.md. If it's not in the manual, the instrument doesn't do it.
